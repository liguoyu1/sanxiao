## 三、核心算法设计要点

### 3.1 六边形坐标系统

```
使用轴向坐标(Axial Coordinates, q, r):
- 六邻域: (q+1,r), (q-1,r), (q,r+1), (q,r-1), (q+1,r-1), (q-1,r+1)
- 距离: max(|dq|, |dr|, |dq+dr|)
- 棋盘半径R: 容纳 ~3R²+3R+1 个格子
```

### 3.2 圈层判定算法（核心）

```
闭环圈层判定 = 在交换后的棋盘上:
1. 对每个同色碎片集合，构建邻接图
2. 查找闭环（Cycle Detection）:
   - 三角: 长度为3的环 → 基础消除
   - 四边: 长度为4的环 → 进阶消除
   - 六边: 长度为6的环 → 全屏共振
3. 嵌套检测: 检查某闭环的几何中心是否在另一闭环内部
```

### 3.3 共振连锁算法

```
1. 从消除圈层的每个碎片出发
2. BFS扩散N步（随圈层等级递增）
3. 扩散范围内的同色碎片标记为「共振激活」
4. 激活后的碎片重新判定是否形成新圈层
5. 若有新圈层 → 递归触发消除+扩散
```

## 四、风险与缓解

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| 圈层判定算法复杂度高 | 大棋盘可能卡顿 | 限制棋盘半径≤5，判定只在交换后执行 |
| 共振连锁失控（无限消除） | 游戏崩溃 | 最大连锁深度限制5层，超限强制结算 |
| 六边形渲染性能 | 低端机掉帧 | 使用Draw Call合并，碎片用SpriteBatch |
| 小游戏包体限制 | 审核不通过 | 纹理压缩、分包加载、远程资源 |
| 玩法太新无用户认知 | 留存低 | 动态教程引导，前3关极低难度 |
